# Headless & CI Mode

Claude Code can run non-interactively — perfect for CI/CD pipelines, scripts, and automation.

## Basic usage

```bash
claude -p "Your prompt here"
# or
claude --print "Your prompt here"
```

## Output formats

| Format | Flag | Use case |
|---|---|---|
| Plain text | `--output-format text` | Human-readable scripts |
| JSON | `--output-format json` | Structured data, piping |
| Streaming JSON | `--output-format stream-json` | Real-time processing |

### JSON output structure

```json
{
  "type": "result",
  "subtype": "success",
  "result": "The output text from Claude",
  "session_id": "abc123",
  "cost_usd": 0.0042,
  "num_turns": 3
}
```

## Limiting turns

Prevent runaway agents with `--max-turns`:

```bash
claude -p "Fix all lint errors" --max-turns 10
```

If the task isn't complete after 10 turns, Claude exits with a non-zero code.

## Restricting tools

For safety in CI, limit which tools Claude can use:

```bash
# Read-only analysis
claude -p "Summarize this codebase" \
  --allowedTools "Read,Glob,Grep"

# Allow only specific bash commands
claude -p "Run tests and report results" \
  --allowedTools "Read,Bash(npm test*)"
```

## Skip confirmations

```bash
claude -p "..." --dangerously-skip-permissions
```

<Callout type="warning">
  Only use `--dangerously-skip-permissions` in sandboxed environments like Docker containers. Never on your development machine or shared infrastructure.
</Callout>

## CI/CD integration examples

### GitHub Actions

```yaml
name: Claude Code Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Review PR
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          DIFF=$(git diff origin/main...HEAD)
          claude -p "Review this PR diff and output a JSON object with 'summary', 'issues', and 'suggestions' keys: $DIFF" \
            --output-format json \
            --max-turns 5 \
            --allowedTools "Read,Glob,Grep"
```

### Pre-commit hook

```bash
#!/bin/bash
# .git/hooks/pre-commit

STAGED=$(git diff --cached --name-only --diff-filter=ACM | grep -E '\.(ts|tsx|js|jsx)$')

if [ -z "$STAGED" ]; then
  exit 0
fi

echo "Running Claude security review on staged files..."

RESULT=$(claude -p "Check these files for security vulnerabilities: $STAGED. Output 'PASS' if none found, or 'FAIL: <reason>' if issues exist." \
  --allowedTools "Read" \
  --output-format text \
  --max-turns 3)

if echo "$RESULT" | grep -q "^FAIL"; then
  echo "Security check failed: $RESULT"
  exit 1
fi

exit 0
```

### Shell script pipeline

```bash
#!/bin/bash

# Generate a changelog from recent commits
COMMITS=$(git log --oneline -20)

claude -p "Generate a user-facing changelog from these git commits. Group by type (Features, Fixes, Internal). Use markdown: $COMMITS" \
  --output-format text > CHANGELOG_DRAFT.md

echo "Changelog written to CHANGELOG_DRAFT.md"
```

## Exit codes

| Code | Meaning |
|---|---|
| `0` | Success |
| `1` | General error |
| `2` | Max turns reached without completion |
| `3` | Authentication error |
