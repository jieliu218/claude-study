# 故障排查

## 身份验证错误

### `ANTHROPIC_API_KEY not set`

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

添加到 `~/.zshrc` 或 `~/.bashrc` 以持久保存。

### `Invalid API key`

- 检查密钥是否含有多余的空格或换行符
- 在 console.anthropic.com 验证密钥是否有效
- 确认没有将项目 API 密钥用于需要个人密钥的场景

## 速率限制

### `429 Too Many Requests`

已达到 API 速率限制。解决方案：

1. 等待后重试（Claude Code 会自动重试）
2. 在 console.anthropic.com 升级 API 套餐
3. 使用 `--max-turns` 限制会话长度

## 上下文窗口问题

### `Context window exceeded`

对话内容已过大。解决方案：

```
/compact         — 压缩历史以释放上下文
/clear           — 清除历史（会丢失上下文）
```

或使用 `--continue` 开始新会话，携带摘要继续。

在配置中启用自动压缩：

```json
{
  "autoCompact": true
}
```

## 工具权限错误

### `Tool not allowed: Bash`

Claude 尝试使用已被拒绝的工具。检查 `settings.json`：

```json
{
  "permissions": {
    "allow": ["Read", "Glob", "Grep", "Write", "Edit", "Bash"]
  }
}
```

### 钩子阻止了工具调用

`PreToolUse` 钩子以代码 `2` 退出。使用 `--verbose` 查看钩子输出：

```bash
claude --verbose
```

## MCP 问题

### 服务器启动失败

```bash
# 独立测试服务器
npx -y @modelcontextprotocol/server-filesystem /tmp
```

常见原因：
- 缺少环境变量（API 密钥）
- Node.js 版本不兼容
- 网络防火墙阻止 SSE 连接

### 工具未出现

在会话中使用 `/mcp` 检查服务器状态。若服务器显示为 `error`：

1. 查看服务器日志：`/mcp logs`
2. 重启服务器：`/mcp restart`
3. 检查 `settings.json` 中的 `command` 和 `args`

## 性能问题

### 响应延迟高

- 切换到更快的模型：`--model claude-haiku-4-5`
- 减少上下文：`/compact` 或开启新会话
- 检查网络连接

### Claude 使用了太多轮次

在提示词中明确范围：

```
不好：  "修复所有 Bug"
更好：  "只修复 src/api.ts 第 42 行的 TypeError"
```

使用 `--max-turns` 强制限制：

```bash
claude -p "..." --max-turns 5
```
